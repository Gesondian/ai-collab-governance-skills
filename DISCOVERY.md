# Discovery Plan

This project is new. Discovery should prioritize useful feedback before broad directory submissions.

The strongest positioning is **Hermes Loop Engineering**: a human-in-the-loop self-improvement loop for AI-assisted software delivery governance.

It is not just a checklist for preventing mistakes. It is a way to turn repeated UAT reopens, weak evidence, overbroad acceptance, and owner-boundary mistakes into reusable skills, templates, examples, and forward tests.

## Current Best Channels

1. Share the launch post in communities where people discuss AI coding agents, UAT, QA, and software delivery.
2. Ask for feedback from teams using agentic coding tools in real delivery workflows.
3. Collect concrete failure cases where an agent accepted too much, fixed too early, routed ownership too broadly, or hardened a rule too quickly.
4. Use that feedback to improve examples, templates, skills, and forward tests through the Hermes loop.
5. Track whether an artifact change changes later agent behavior; do not count documentation edits alone as self-improvement.

## Awesome List Submission Readiness

Some agent-skill awesome lists accept new community submissions, but the largest lists may require signs of real community usage.

For example, `VoltAgent/awesome-agent-skills` asks for:

- a public repository,
- working skill documentation,
- a short description,
- links that work,
- and real community usage before submission.

Because this project is newly published, submit to that list only after there is at least some external usage signal, feedback, or stars.
