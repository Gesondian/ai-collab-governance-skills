# Discovery Plan

This project is new. Discovery should prioritize useful feedback before broad directory submissions.

## Current Best Channels

1. Share the launch post in communities where people discuss AI coding agents, UAT, QA, and software delivery.
2. Ask for feedback from teams using agentic coding tools in real delivery workflows.
3. Collect concrete failure cases where an agent accepted too much, fixed too early, or routed ownership too broadly.
4. Use that feedback to improve examples and forward tests.

## Awesome List Submission Readiness

Some agent-skill awesome lists accept new community submissions, but the largest lists may require signs of real community usage.

For example, `VoltAgent/awesome-agent-skills` asks for:

- a public repository,
- working skill documentation,
- a short description,
- links that work,
- and real community usage before submission.

Because this project is newly published, submit to that list only after there is at least some external usage signal, feedback, or stars.
